Стек (Stack)
Принцип: LIFO (Last In, First Out) - "последним пришёл, первым ушёл"

Аналогия: Стопка тарелок или история браузера (кнопка "Назад")

Основные операции:
- push(item) - добавить элемент на вершину
- pop() - удалить и вернуть элемент с вершины
- peek() - посмотреть элемент на вершине (без удаления)
- empty() - проверить, пуст ли стек

Временная сложность: O(1) для всех операций
Очередь (Queue)
text
Принцип: FIFO (First In, First Out) - "первым пришёл, первым ушёл"

Аналогия: Очередь в кассу или очередь печати документов

Основные операции:
- enqueue(item) - добавить элемент в конец очереди
- dequeue() - удалить и вернуть элемент из начала очереди
- peek() - посмотреть первый элемент (без удаления)
- empty() - проверить, пуста ли очередь

Временная сложность: O(1) для всех операций (с использованием deque)
Односвязный список (Singly Linked List)

Структура: Динамическая цепочка узлов, где каждый узел содержит:
- Данные (значение)
- Ссылку на следующий узел (next)

Основные операции:
- append(value) - добавить элемент в конец
- prepend(value) - добавить элемент в начало
- insert(idx, value) - вставить элемент по индексу
- remove_at(idx) - удалить элемент по индексу
- remove(value) - удалить первый элемент с заданным значением
- Итерация по элементам

Временная сложность:
- Добавление/удаление в начало: O(1)
- Добавление в конец (с tail): O(1)
- Доступ по индексу: O(n)
- Поиск по значению: O(n)
Практическая часть
Реализованные классы
1. Класс Stack
Реализация стека на основе встроенного списка Python.
2. Класс Queue
Реализация очереди на основе deque из модуля collections.
3. Класс SinglyLinkedList
Реализация односвязного списка с классами Node и SinglyLinkedList.
Особенности реализации
- Поддержка метода __len__ для получения размера структуры
- Реализация строкового представления через __repr__
- Поддержка итерации (__iter__) для односвязного списка
- Обработка граничных случаев (пустые структуры, неверные индексы)
- Корректное обновление head и tail в связном списке

Тестирование
Для каждого класса разработаны тесты, проверяющие:

1. Инициализация - создание пустой структуры
2. Добавление элементов - операции push/enqueue/append/prepend
3. Удаление элементов - операции pop/dequeue/remove/remove_at
4. Просмотр элементов - операции peek
5. Обработка ошибок - исключения при операциях с пустыми структурами
6. Граничные указатели - корректность head и tail в связном списке
7. Итерация - корректность обхода элементов
8. Размер структуры - корректность подсчёта элементов

Применение
Стек
- Управление историей операций (Ctrl+Z)
- Проверка корректности скобок в выражениях
- Обход графов в глубину (DFS)
- Вызов функций в программировании (call stack)
Очередь
- Очередь задач для обработки
- Буферизация данных
- Обход графов в ширину (BFS)
- Очередь печати документов
Односвязный список
- Динамические коллекции с частыми вставками/удалениями в начале
- Реализация стеков и очередей
- История навигации в браузерах
- Управление памятью в операционных системах
